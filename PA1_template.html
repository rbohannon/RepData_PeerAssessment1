Imputing missing values

On visually inspecting the data, it looks like the only attribute missing data is steps. We confirm this by checking each column for NA values.

apply(activity, 2, function(x) any(is.na(x)))

##    steps     date interval 
##     TRUE    FALSE    FALSE

Now that we are certain the only column that contains NA values is the first column (steps), we count how many NAs there are in that column.

NAcount <- sum(is.na(activity[, 1]))
  • The total number of rows containing NAs is 2,304.


We want to fill in the missing values, but we also want to preserve the original dataset. We make a copy of the data.

activityNoNA <- activity


Strategy

To fill in the missing values, we loop over the duplicate dataset. For each row, if steps contains NA, we retrieve the interval from that same row. Then from the stepsMeanPerInterval we created earlier we extract the steps from the row that has the corresponding interval. Lastly, we replace the NA in the duplicate dataset with the mean value retrieved from stepsMeanPerInterval.

# loop over duplicate dataset and replace NAs with
# mean for the corresponding interval
for (i in 1:nrow(activityNoNA)) {

        # if current row is missing value
        if (is.na(activityNoNA[i, 1])) {

                # get interval
                currInterval <- activityNoNA[i, 3]

                # retrieve the mean for the interval
                currMean <- stepsMeanPerInterval[
                        which(stepsMeanPerInterval[, 1] == currInterval), 
                        2]

                # replace NA with mean
                activityNoNA[i, 1] <- currMean
        } # end if
} # end for

Are there differences in activity patterns between weekdays and weekends?

To examine the activity patterns for weekdays and weekends, it will be helpful to have a new factor to designate which dates are week days and which are weekend days.

First we create a function to determine the type of day (weekday or weekend) for a specific date. The function takes a date as an argument and checks the name of the day of the week. If the day name is “Saturday” or “Sunday”, “weekend” is returned. Otherwise, “weekday” is returned.

DayType <- function(x) {

        switch(weekdays(x),
               "Saturday" = ,
               "Sunday" = "weekend",
               "weekday")
}

To create the factor, we start by creating an empty vector. Then we loop over the duplicate dataset and determine if each date is a weekday or weekend. As each day type is determined, it is added to the vector. When complete, the vector is added to the dataset as a factor.

dayType <- c()

for (i in 1:nrow(activityNoNA)) {

        dayType <- c(dayType, DayType(as.Date(activityNoNA[i, 2], "%Y-%m-%d")))
}

activityNoNA$dayType <- factor(dayType)

To construct the panel plot of the average number of steps taken, averaged across all weekday days or weekend days, we calculate the means. Then, we assign values we want to plot to variables to make the code more human readable and build the plot using xyplot() from the lattice package.

stepsMeanPerIntervalPerDayType <- aggregate(steps ~ interval + dayType, 
                                            data=activityNoNA, mean)

steps <- stepsMeanPerIntervalPerDayType$steps
interval <- stepsMeanPerIntervalPerDayType$interval
dayType <- stepsMeanPerIntervalPerDayType$dayType

library(lattice)
xyplot(steps ~ interval | dayType, 
       layout=c(1, 2), 
       type="l",
       xlab="Interval",
       ylab="Number of Steps")
